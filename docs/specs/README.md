# Specifications

Formal product or technical specifications (PRDs, technical specs, API contracts) can live here. Use **lowercase-with-hyphens** filenames (e.g. `feature-name-spec.md`, `api-contract.md`).

In this template, **scope and acceptance criteria come from GitHub Issues** (see `.github/ISSUE_TEMPLATE/` in the repo root); you do not need a spec file per feature unless you want a written document. Add files here when you need a single source of truth for a larger feature, integration contract, or cross-team spec. Link to or reference the relevant issue from the spec when both exist.
