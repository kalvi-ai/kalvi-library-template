# Project review rubric (optional extension)

The **project-review** skill builds its rubric from project config, principles, and built-in criteria. This file is **optional**: if present, the skill merges the criteria below into the rubric so you can add project-specific or extra dimensions without editing the skill.

## How to use

- Keep each criterion as a short line or bullet.
- The skill will assess each and report **Pass** / **Partial** / **Fail** with evidence and an optional suggestion.
- Remove or comment out criteria that do not apply to your project.

## Optional criteria (add or customize)

- **Accessibility**: Public APIs or UIs documented for accessibility where relevant.
- **Performance**: No obvious performance anti-patterns; benchmarks or profiling notes if applicable.
- **Observability**: Logging, metrics, or error reporting approach documented or present.
- **API stability**: Public API surface documented; versioning or compatibility policy if applicable.
- **Onboarding**: README or CONTRIBUTING explains how to set up and run the project.

Add your own rows below; the skill will treat them as additional rubric rows.

- **Template escape (if derived from this template)**: After bootstrap, no remaining "this template" or `kalvi_library_template` in user-facing docs and skills; wording and paths refer to the new project.
